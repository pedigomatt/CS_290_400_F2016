This Guide will walk you through the steps and share point out specific points to use the API to filter for desired news stories.

AlchemyData News indexes over 250k english news sites every day making the information available for AI scanning. Historical searches can be done back 60days.

This guide will skip over the obvious stuff about setting up the API key. The directions for this are fairly obvious,so they will only be mentioned at the beginning.

  2. The next step is to review the details of the get request. Currently, this API only allows get request but soon it will allow push requests as well.
    There are several parts to the API request that tells the application what type of news articles you are looking for and what values you want back in response.
    The AlchemyData News uses a rest API and the following lists out the components to the API request.
    These different parts are strung together with an '&' symbol.
    • REST API Endpoint: https://gateway-a.watsonplatform.net/calls/data/GetNews?
    • apikey=YOUR_APP_APIKEY (replace YOUR_APP_APIKEY with the actual API key from the bluelink website)
    • outputMode - can be either JSON format or XML. Example would be 'outputMode=JSON'
    • start - identifies the date and time that you want to start your search from. If wanting to do the previous 24hrs, it would be 'start=now-1d'
    • end - identifies the date and time that you want to end your search at. Example would be 'end=now'
    • count - identifies the number of articles that are wanted to be returned. Could be an integer with any number of values.
    • Targeted Searches - There are 6 different types of search criteria. They can be applied together or seperate.
      • q.enriched.url.enrichedTitle.taxonomy.taxonomy_.label - Taxonomy - Specific to a list of titles that are cataloged by the app.Can also be narrowed down to further subsets of different titles.
      • q.enriched.url.concepts.concept.text - Concepts - Can be any string value that wants to be searched for.
      • q.enriched.url.enrichedTitle.keywords.keyword.text - Keywords - The AlchemyData news app automatically searches out keywords. Will return any articles that have matching keywords.
      • q.enriched.url.enrichedTitle.entities.entity.text - Entity - This specifies the 2 strings - a text and a type. This allows for differentiation between 2 different things that may have the same name.
      • q.enriched.url.enrichedTitle.relations.relation - Relation - Looks for relationships in articles which allows for more precise searches. Usually includes 2 nouns and a verb.
      • q.enriched.url.enrichedTitle.docSentiment - Sentiment - Specifies the mood of the article. Such as happy or angry. Can further refine the search to look for a certain emotion around a specific topic.
    • return - specifies what type of return values are needed.There are over 250 different types of values that can be returned.Return types are seperated by a comma. Common return types are:
      • enriched.url.title - Returns the title of the article
      • enriched.url.entities.entity.text - Returns a portion of the text in the article
      • enriched.url.author - Returns the name of the author
      • enriched.url.url - Returns the url to the full article
    • rank - specifies if the returned values should be ranked based on feedback as high, medium or low
    • return - specifies what type of return values are needed.There are over 250 different types of values that can be returned.
